  - `preview_image` (string, optional): Base64-encoded image (e.g., PNG, JPEG) used as the file's preview in Slack.
  - `indexable_file_contents` (string, optional): Plain text content of the file, indexed by Slack for search.

### SLACKBOT_CREATE_A_REMINDER

**MCP Tool**: `mcp__composio__SLACKBOT_CREATE_A_REMINDER`

Creates a Slack reminder with specified text and time; time accepts Unix timestamps, seconds from now, or natural language (e.g., 'in 15 minutes', 'every Thursday at 2pm').

**Parameters:**
  - `text` (string, required): The textual content of the reminder message.
  - `time` (string, required): Specifies when the reminder should occur. This can be a Unix timestamp (integer, up to five years from now), the number of seconds until the reminder (integer, if within 24 hours, e.g., '300' for 5 minutes), or a natural language description (string, e.g., "in 15 minutes," or "every Thursday at 2pm", "daily"). 
  - `user` (string, optional): The ID of the user who will receive the reminder (e.g., 'U012AB3CD4E'). If not specified, the reminder will be sent to the user who created it. NOTE: Setting reminders for other users is no longer supported for user tokens - only bot tokens can set reminders for other users.
  - `team_id` (string, optional): Encoded team id. Required if using an org-level token to specify which workspace the reminder should be created in.
  - `recurrence` (object, optional): Recurrence settings for a reminder.

### SLACKBOT_CREATE_A_USER_GROUP

**MCP Tool**: `mcp__composio__SLACKBOT_CREATE_A_USER_GROUP`

Creates a new User Group (often referred to as a subteam) in a Slack workspace.

**Parameters:**
  - `name` (string, required): Unique name for the User Group. Must be unique among all User Groups in the workspace.
  - `handle` (string, optional): Unique mention handle. Must be unique across channels, users, and other User Groups. Max 21 chars; lowercase letters, numbers, hyphens, underscores only.
  - `team_id` (string, optional): Encoded team ID where the User Group should be created. Required if using an org token. Will be ignored if the API call is sent using a workspace-level token.
  - `channels` (string, optional): Comma-separated encoded channel IDs for default channels, suggested when mentioning or inviting the group.
  - `description` (string, optional): Short description for the User Group.
  - `include_count` (boolean, optional): Include the User Group's user count in the response. Server defaults to `false` if omitted.
  - `enable_section` (boolean, optional): Configure this user group to show as a sidebar section for all group members. Only relevant if group has 1 or more default channels added.
  - `additional_channels` (string, optional): Comma-separated encoded channel IDs for which the User Group can custom add usergroup members to.

### SLACKBOT_CUSTOMIZE_URL_UNFURL

**MCP Tool**: `mcp__composio__SLACKBOT_CUSTOMIZE_URL_UNFURL`

Customizes URL previews (unfurling) in a specific Slack message using a URL-encoded JSON in `unfurls` to define custom content or remove existing previews.

**Parameters:**
  - `ts` (string, optional): Timestamp of the message to customize URL unfurling for. Must be provided with `channel`, or alternatively provide `unfurl_id` and `source` together.
  - `source` (string, optional): Link source: either 'composer' or 'conversations_history'. Must be provided with `unfurl_id`.
  - `channel` (string, optional): Channel, private group, or DM channel to send message to. Can be an encoded ID, or a name. Must be provided with `ts`, or alternatively provide `unfurl_id` and `source` together.
  - `unfurls` (string, optional): JSON string mapping URLs to custom unfurl content (Slack attachment format or blocks). Pass as a plain JSON string (not URL-encoded). To remove an existing unfurl, provide an empty object for that URL.
  - `metadata` (string, optional): JSON object with 'entities' field providing Work Object array. Either `unfurls` or `metadata` is required. Pass as a JSON string.
  - `unfurl_id` (string, optional): Link ID to unfurl. Must be provided with `source`. Alternative to using `channel` and `ts` parameters.
  - `user_auth_url` (string, optional): URL-encoded custom URL for user authentication with your app to enable unfurling. Used when `user_auth_required` is true.
  - `user_auth_blocks` (string, optional): JSON array of structured blocks (URL-encoded) sent as ephemeral authentication invitation. Alternative to `user_auth_message` for richer formatting. Used when `user_auth_required` is true.
  - `user_auth_message` (string, optional): Ephemeral message text prompting user authentication with your app for domain-specific unfurling. Used when `user_auth_required` is true and authorization is pending.
  - `user_auth_required` (boolean, optional): Set to `true` if user authentication is required to unfurl links for a domain, enabling an authentication flow using `user_auth_url` and `user_auth_message`.

### SLACKBOT_DOWNLOAD_FILE

**MCP Tool**: `mcp__composio__SLACKBOT_DOWNLOAD_FILE`

Tool to download Slack file content and convert it to a publicly accessible URL. Use when you need to retrieve and download files that have been shared in Slack channels or conversations.

**Parameters:**
  - `file` (string, required): ID of the file to download. This is a required field. File IDs start with 'F' followed by alphanumeric characters (e.g., 'F123ABCDEF0').
  - `page` (integer, optional): Page number of comment results to retrieve. Used for comment pagination. Slack's default is 1 if not provided. `cursor`-based pagination is generally preferred.
  - `count` (integer, optional): Number of comments to retrieve per page. Used for comment pagination. Slack's default is 100 if not provided.
  - `limit` (integer, optional): The maximum number of comments to retrieve. This is an upper limit, not a guarantee of how many will be returned. Primarily used for comment pagination.
  - `cursor` (string, optional): Pagination cursor for retrieving comments. Set to `next_cursor` from a previous response's `response_metadata` to fetch the next page of comments. Essential for navigating through large sets of comments.

### SLACKBOT_EDIT_CANVAS

**MCP Tool**: `mcp__composio__SLACKBOT_EDIT_CANVAS`

Edits a Slack Canvas with granular control over content placement. Supports replace, insert (before/after/start/end) operations for flexible content management.

**Parameters:**
  - `canvas_id` (string, required): The unique identifier of the canvas to edit
  - `operation` (string, optional): Type of edit operation: 'replace' (replaces entire canvas or specific section if section_id provided), 'insert_after' (inserts content after section_id), 'insert_before' (inserts content before section_id), 'insert_at_start' (prepends content to beginning), 'insert_at_end' (appends content to end), 'delete' (deletes specific section by section_id), 'rename' (renames canvas title using title_content)
  - `section_id` (string, optional): Section ID for targeted operations. Required for: 'insert_after', 'insert_before', 'delete'. Optional for: 'replace' (if omitted, replaces entire canvas). Not used for: 'insert_at_start', 'insert_at_end'. Use canvases.sections.lookup method to get section IDs from existing canvas.
  - `title_content` (object, optional): The new title for the canvas in markdown format. Required only for 'rename' operation. Supports markdown format including emojis (e.g., ':white_check_mark:').
  - `document_content` (object, optional): The content to add/replace in Slack's document format. Required for all operations except 'delete' and 'rename'. Use canvases.sections.lookup to find section IDs for targeted operations.
